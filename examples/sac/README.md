## SAC 算法概述

### 核心思想

SAC（Soft Actor-Critic）是一种 **最大熵强化学习** 算法。与传统 RL 只追求累积奖励不同，SAC 的目标函数同时最大化 **期望回报** 和 **策略的熵**：

```
J(π) = E[Σ_t γ^t (r_t + α · H(π(·|s_t)))]
```

其中 α（温度参数）控制"获取奖励"与"保持探索"之间的权衡。

### 算法演化

| 版本 | 论文 | α 处理方式 | Action 类型 |
|------|------|-----------|------------|
| SAC v1 (2018a) | *Soft Actor-Critic: Off-Policy Maximum Entropy Deep RL...* | **手动固定** | 连续 |
| SAC v2 (2018b) | *Soft Actor-Critic Algorithms and Applications* | **自动调节**（约束优化） | 连续 |
| SAC-Discrete (2019) | *Soft Actor-Critic for Discrete Action Settings* | 自动调节 | 离散 |
| Hybrid SAC (2019) | *Discrete and Continuous Action Representation for Practical RL...* | 自动调节（α_d + α_c 分离） | 混合 |

### Actor Loss 公式

本项目的 SAC 示例统一使用 **原始公式**（以 `log_prob` 为核心构建），而非 `entropy()` 风格：

```
L = mean(Σ_a π(a) × (α·log π(a) - Q(a)))
```

等价于 `L = -(E_π[Q] + α·H(π))`，但这种写法有两个关键优势：

1. **统一性**：通过"哑值填充"可以让 Discrete / Continuous / Hybrid 三种 Action 类型走同一段代码（参见 [rustRL](https://github.com/dbsxdbsx/rustRL) 的设计）
2. **简洁性**：计算图中节点更少（无需 entropy 内部的 Negate、Multiply、Sum 节点）

## 核心概念：熵（Entropy）

### 什么是熵？

熵衡量的是 **"你有多犹豫"** —— 一个概率分布的不确定程度。

- 熵高 → 策略更"犹豫"/更随机 → 动作选择更分散
- 熵低 → 策略更"果断"/更确定 → 动作选择更集中

这一语义对离散、连续、混合 Action **都成立**，与 Action 类型本身无关。

### 单位

熵的"单位"取决于对数的底数。SAC 使用自然对数（base e），单位是 **nat**（奈特），本质上是无量纲的。

> **补充**：这里说的是 **信息熵**（Shannon Entropy），与物理学中的热力学熵（如薛定谔在《生命是什么》中讨论的）概念相通（"熵越大 = 越无序"），但语境不同。

### 离散 vs 连续：熵的本质差异

这是理解 SAC 参数设置的**最关键前提**。

#### 离散动作的熵（Shannon 熵）

公式：`H = -Σ p(x) · ln(p(x))`

因为每个 p(x) 是**概率**，取值范围 [0, 1]，所以 ln(p(x)) ≤ 0，每一项 -p·ln(p) ≥ 0。
因此 **离散熵上下界都是固定的，且两端都能精确达到**。

以 CartPole（2 个动作：左推/右推）为例：

```
[0.50, 0.50]  →  完全犹豫，随机乱选     → H = 0.693（= ln(2)，上界）
[0.70, 0.30]  →  有点偏好，但还有犹豫   → H ≈ 0.611
[0.89, 0.11]  →  比较确定，偶尔试试别的 → H ≈ 0.347
[0.99, 0.01]  →  非常确定，几乎不犹豫   → H ≈ 0.056
[1.00, 0.00]  →  完全确定，零犹豫       → H = 0（下界，可精确达到）
```

**离散熵的范围：[0, ln(K)]**，其中 K 是动作数量。就像"选门"——你可以 100% 确定只走 1 号门，其他门完全封死，实现零犹豫（H = 0）。

#### 连续动作的熵（微分熵）

公式：`h = -∫ f(x) · ln(f(x)) dx`

这里 f(x) 是概率 **密度**（不是概率）。关键区别：**密度可以大于 1**。

例如 σ = 0.1 的 Gaussian，峰值密度 f(0) = 1/(0.1 × √(2π)) ≈ 3.99 > 1。当密度 > 1 时，ln(f) > 0，使得微分熵可以为 **负数**。

以 1 维 Gaussian（动作范围 [-1, 1]）为例：

```
σ = 1.0   →  散布很广，弹孔满地  → h ≈ +1.42
σ = 0.3   →  比较集中           → h ≈ +0.22
σ = 0.09  →  很精确             → h ≈ -1.0   ← SAC v2 默认 target 在此
σ = 0.01  →  几乎确定性         → h ≈ -3.2
```

**微分熵没有固定的上界或下界（范围为 (-∞, +∞)），且两端都只能趋近而不能精确达到。**

#### 为什么连续熵可以为负？

这不仅是数学定义的差异，而是有本质原因：

- **离散熵**有天然零点——"完全确定（只有一个选择）"时 H = 0。
- **微分熵没有天然零点。** 它本质上是与隐含的"单位密度均匀分布"做比较：h > 0 表示"比参考更分散"，h < 0 表示"比参考更集中"。负值不代表"负的随机性"。

#### 能否达到"完全确定"？

| 类型 | 如何实现"完全确定" | 熵 = 0 可达？ |
|------|-------------------|-------------|
| **离散** | 概率设为 [1, 0, 0, ...] | **可以**精确等于 0 |
| **连续** (Gaussian) | σ → 0（趋向 Dirac δ 函数） | **不行**，h → -∞，永远无法精确达到某个有限值 |

Gaussian 的尾巴是无限延伸的，不管 σ 多小，理论上每个点的概率密度都 > 0。所以连续策略（基于 Gaussian）**永远无法变成 100% 确定性的**——就像"扔飞镖"，你可以把圈越缩越小，但 Gaussian 告诉你飞镖理论上可以落在任何地方。

#### 重要：连续熵依赖动作范围

同一个 Gaussian，缩放动作范围后熵会变化：

```
范围 [-1, 1]，σ = 0.1  → h ≈ -0.88
范围 [-10, 10]，σ = 1.0 → h ≈ +1.42  （同样的"相对精度"，熵增加了 ln(10) ≈ 2.3）
```

分布的"形状"完全一样，只是坐标轴拉伸了，熵就变了。**这就是为什么 SAC 几乎都要把连续动作归一化到 [-1, 1]** —— 在这个标准化的尺度下，Target Entropy 才有统一的参照基准。

#### 熵的范围总结

```
离散：H ∈ [0, ln(K)]    ← 两端都能精确达到
连续：h ∈ (-∞, +∞)      ← 两端都只能趋近，不能精确达到
```

## 核心概念：Alpha（α，温度参数）

### α 的含义

| α 值 | 行为 |
|------|------|
| α → 0 | 完全贪婪，退化为普通 RL（忽略 entropy，只看 Q 值） |
| α 适中 | 平衡探索与利用（SAC 的优势所在） |
| α → ∞ | 完全随机，忽略奖励（最大化 entropy，等于随机策略） |

### 自动调节机制（SAC v2）

SAC v2 将 entropy 作为 **约束** 而非目标，通过拉格朗日对偶转化实现自动调节：

```
约束：E[-log π(a|s)] ≥ H_target
对偶变量（Lagrange multiplier）：α

更新规则（对 log α 做梯度下降）：
  ∇_{log α} L(α) = α · (H(π) - H_target)
```

直觉如恒温器：
- **实际熵 < 目标熵**（策略太确定了）→ α 增大 → 强制更多探索
- **实际熵 > 目标熵**（策略够随机了）→ α 减小 → 允许更多利用

### α 的参数化与安全边界

α 被参数化为 `α = exp(log_α)`，确保 α > 0。理论上 α 没有上下界。

如果 Target Entropy 设置合理，α 通常会在自动调节下收敛到一个合理的范围（例如 0.01 ~ 2.0）。**但仍建议保留 clamp 保护**，原因是 α 的更新存在正反馈风险：

```
策略突然坍缩 → H 骤降到远低于 target
→ 梯度 = α × (H - target)，大负数
→ log_α 大幅增加 → α = exp(log_α) 变更大
→ 下一步梯度更大（因为梯度里乘了 α）
→ 可能指数级膨胀 💥
```

正常情况下，α 增大会推高 entropy bonus，entropy 回升形成负反馈。但如果 Q 网络同时出现估计偏差，α 可能在恢复前已经冲到离谱的值。

因此，工业级实现通常对 `log_α` 做 clamp 保护——成本几乎为零，就像开车系安全带：

```rust
log_alpha = log_alpha.clamp(-20.0, 2.0);
// → α ∈ [~2e-9, ~7.4]
```

这个范围对离散/连续/混合 Action 都适用，正常训练中几乎不会触发。

## Target Entropy：如何设置？

**Target Entropy = "动作空间的自然尺度" × "期望的随机性比例"。** 两者缺一不可——脱离动作空间的锚点，系数设得再高或再低都没有意义。

### 离散动作

```
target = c × ln(K)
```

其中 K 是动作数量，ln(K) 是该动作空间的熵"天花板"（均匀分布时的最大熵）。

| c 值 | 含义 | 适用场景 |
|------|------|---------|
| 0.98 | 几乎跟随机一样犹豫 | 探索非常重要、奖励稀疏的环境 |
| **0.5** | **该果断时果断，保持试探** | **大多数标准离散环境（推荐起步值）** |
| 0.1 | 几乎确定性选动作 | 最优策略非常清晰的简单环境 |

例如 CartPole（K=2, c=0.5）：`target = 0.5 × ln(2) ≈ 0.347`，对应约 [0.89, 0.11] 的概率分布——"89% 把握选某个方向，但保留 11% 的反向试探"。

### 连续动作

```
target = -d × f
```

其中 d 为动作维度，f 通常取 1.0（SAC v2 默认值 `-d`），**前提是动作已归一化到 [-1, 1]**。

| f 值 | 含义 |
|------|------|
| 0.5 | 较宽松，允许更大随机性 |
| **1.0** | **标准值（SAC v2 默认，推荐起步值）** |
| 1.5 | 较严格，要求更高精度 |

**`target = -d` 的直觉含义**：在 [-1, 1] 归一化的动作空间中，每个维度的收敛 σ ≈ 0.089，即动作的抖动范围约为 ±9%。通俗来说："我很确定该怎么做，但保留大约 ±9% 的微调空间"。

| 动作维度 d | target entropy | 每维对应 σ |
|-----------|---------------|-----------|
| 1 | -1 | ≈ 0.089 |
| 3 | -3 | 每维 ≈ 0.089 |
| 20 | -20 | 每维 ≈ 0.089 |

### 混合动作（Hybrid）

离散和连续部分 **必须分别设置** 独立的 α 和 target entropy：

```
entropy bonus = α_d · H(π_d) + α_c · Σ_{a^d} π(a^d|s) · H(π_c|a^d)
```

原因：离散和连续的熵量级差异很大，用同一个 α 会导致一方"淹没"另一方（[Delalleau et al., 2019]）。各自的 `log_α` clamp 范围可以统一（如 [-20, 2]），实际值由各自的 target entropy 驱动。

### 拿到新项目时的速查清单

```
1. 判断动作类型
   ├─ 离散 → target = 0.5 × ln(K)
   ├─ 连续 → 归一化到 [-1, 1]，target = -d
   └─ 混合 → 分别设 target_d 和 target_c

2. Alpha 初始化
   └─ log_alpha = 0（即 α = 1.0），中性起点

3. Alpha 安全边界
   └─ log_alpha clamp 到 [-20, 2]（成本为零的安全带）

4. 训练观察
   ├─ α 正常范围约 0.01 ~ 2.0
   ├─ α 单调下降 → 正常（初始 entropy 高于 target）
   ├─ α 先降后升 → 正常（策略一度过于确定后自我修正）
   └─ α 持续上升到 >5 → 检查 target 是否过高，或策略是否坍缩
```
